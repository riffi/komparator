# AGENTS

## Spec First

В проекте есть две основные спецификации:

- `docs/spec/ui-spec.yaml`
- `docs/spec/data-spec.yaml`

Доработка спецификаций должна выполняться в соответствии с описанием форматов:

- `docs/spec/formats/ui-spec-format-0-0-3.yaml`
- `docs/spec/formats/data-spec-format-0-0-1.yaml`

Любые доработки приложения нужно делать по подходу `spec first`.

Порядок работы:

1. Сначала обновить соответствующие спецификации (`ui spec`, `data spec`).
2. Вносить изменения в спеки строго по правилам и структуре, описанным в файлах форматов.
3. Убедиться, что изменения в спеках описывают новое поведение, структуру данных или интерфейс.
4. Только после этого вносить изменения в код.

Если изменение затрагивает UI и данные одновременно, сначала нужно синхронно обновить обе спецификации, и только потом переходить к реализации.
